# Project Forge Specification

## Mission
Project Forge is the internal engineering system for building Project Salus faster, cleaner, and with less technical debt.

## Core Rule
Build platforms, not point solutions.

## Project Salus Core Engines
1. Mission Engine
2. Memory Engine
3. Knowledge Engine
4. Agent Engine
5. Decision Engine
6. Automation Engine
7. Legacy Engine

## Engineering Rules
- Every module must be modular.
- Every module must be testable.
- Every module must be replaceable.
- Every major action must be observable.
- Every feature must support judgment, execution, security, or legacy.

## Barney Legacy Requirement
Every major feature must answer:
Would this make Kyle's family better off across a 10, 25, 50, or 100 year horizon?

## AI Builder Team
- Chief Architect
- Backend Engineer
- Frontend Engineer
- Security Engineer
- QA Engineer
- Documentation Engineer
- Legacy Architect

## Definition of Done
A feature is not done until:
- Code runs
- API tested
- UI tested if applicable
- Git status reviewed
- Changes committed
- Documentation updated
